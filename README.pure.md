# Pure Blocks

## Flows
- Pure UI blocks receive data via props and emit events via callbacks.

## Requirements
- No side effects (no filesystem, network, IPC).
- Platform-agnostic and reusable across apps.

## UX
- Blocks are self-contained and focus on rendering and local interaction.

## Style
- Keep dependencies limited to UI and type packages.

## Data Models
- Inputs/outputs are defined by block props and events.
